# 新风控系统现状（个人理解）

> 去掉产品内部名称（如法币通道品牌），用通用模块名描述。

## 一、功能架构

链路：**规则配置 → 实时决策 → 人工审核 → 监控告警 → 数据回溯**

```
业务请求 → 实时引擎 / 监控引擎 → 审核中心 / 告警
                ↑
           指标系统（数据源、维度、指标、引擎日志）
```

### 1. 审核中心

统一管理多业务审核任务，例如：

- 链上提现：币种、哈希、地址、数量、手续费、命中规则
- 法币/支付通道出金：渠道、收款信息、汇率、订单号
- 链上充值复核
- 营销活动参与资格

### 2. 风险标签

- 自定义标签：名称、等级、说明、操作人
- 实体绑定：用户 ID 等，记录原因、重复次数、最后标记时间

### 3. 决策引擎

- 场景：按业务划分，灰度、负责人、启停
- 参数：阈值、开关
- 变量库：场景变量、常量、衍生指标

### 4. 监控引擎

- 监控事件：业务事件、旁路比例、兜底策略
- 监控规则：脚本化逻辑（如 Groovy），支持命中规则、告警、策略结果

### 5. 指标系统

数据源接入、维度、指标配置、回溯、引擎日志查询。

## 二、我关注的演进方向（Agent）

| 场景 | 输入 | 输出 | 价值 |
|-----|------|------|------|
| 审核辅助 | 案件 + 标签 + 关联图谱 | 建议 + 摘要 | 缩短人工时长 |
| 策略调优 | 拦截率、误杀、通过率 | 参数建议 + A/B | 自适应阈值 |
| 告警降噪 | 高频告警日志 | 聚合 + 抑制规则 | 减运营疲劳 |

## 三、个人小结

系统已覆盖策略—审核—标签—监控主链路，适合作为后续 Agent 试点的底座；下一步我更想先落地「审核辅助」和「告警降噪」两个 PoC。
