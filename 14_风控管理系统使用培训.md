# 风控管理系统 — 个人使用笔记

> 培训录屏链接已删除；环境 URL 改为占位，实际地址从密码库获取。

## 目的

尽快上手：登录权限、指标系统、实时引擎、监控引擎、审核与看板。

## 登录与权限

1. 入职 **LDAP / 统一身份（IAM）** — 多「站点/矩阵」可能不同 VPN  
2. 页面权限找 **风控产品同事** 开通  
3. 环境示例（占位）：  
   - 测试：`https://risk-admin-***-dev.example/welcome`  
   - 生产：`https://risk-admin-***-prod.example/welcome`  
   - 其他业务线可能有独立后台域名  

## 模块地图（2.0 结构）

| 模块 | 子功能 |
|-----|--------|
| 风控看板 | 首页、告警 |
| 决策引擎 | 场景、参数、变量、条件、动作、函数、节点、决策流、验证 |
| 客服中心 | 法币出金审核、提现审核 |
| 模型平台 | 指标、工具箱、名单、标签、实时/监控引擎、日志、审核中心、用户处置、告警配置、元数据、操作日志、系统管理 |

## 指标系统（我常用）

- **数据来源**：MySQL/Mongo/Kafka 等，配置表名、消费类型、清洗脚本  
- **维度管理**：单维/多维，供指标引用  
- **指标配置**：近线（分钟~天级延迟）、离线（小时~年级延迟）  

## 实时引擎

- 实时事件 + 实时规则 → 策略命中  

## 监控引擎

- 监控事件 + Groovy 规则 → 告警  

## 数据流（简图）

```
业务日志/交易 → 指标 → 实时/监控规则 → 命中 → 审核任务/告警
```

## 个人提示

- 改规则先在**测试环境**验证决策流  
- 动生产参数前看灰度比例和负责人  
- 看不懂命中原因先查**引擎日志**
