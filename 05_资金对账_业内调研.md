# 资金对账 — 业内调研笔记（个人版）

## 一、对账流程（链上 + 账务）

1. **链上核对**：浏览器按币种/地址扫 TXID，防「空充值」。
2. **账务流水**：链上记录 vs 用户账本，「入账 − 出账 = 余额变动」。
3. **热冷划转**：Gas、归集、调拨要可追溯。

## 二、交易所类型（我的对比表）

| 类型 | 代表 | 特点 |
|-----|------|------|
| 综合超大所 | 头部综合所 | 独立风控级对账、存量+增量双跑；包袱重、异常量大 |
| 合规/法币友好 | 美股上市所等 | 总账与链上一致、审计友好；往往 T+1，实时弱 |
| 强衍生品 | 头部衍生品所 | 增量事件强、仓位级敏感；存量视角与财务易割裂 |
| 中小型 | 新所 | 链路短易改；缺独立风控对账时单点风险大 |

## 三、实时 vs 定时

**钱包对账**：热/冷钱包、链上、托管方 — 余额、充提 TX、归集、Gas、冻结 → **资产真实性**

**交易所对账**：撮合 vs 外部所 — 订单、成交、持仓、冻结、盈亏 → **业务真实性**

**财务对账**：钱包 + 三方支付 + 总账 → **会计合规**

## 四、三层不能互相替代

- 交易对得上 ≠ 钱对得上  
- 钱对得上 ≠ 钱包真有钱  
- 钱包有钱 ≠ 账务没问题  

**建议顺序**：交易所（准实时）→ 钱包（真实性）→ 财务（结果校验）

## 五、调研报告摘要（大厂做法）

对账设计共性：**明确业务模型 → 抽象核对模型 → 选方案 → 系统设计**

交易所 PoR/储备证明模块（币安、OKX、bitFlyer、BingX 等）— 我记的重点是「负债快照 + 链上资产 + 覆盖率」，细节见公开 PoR 报告，不写内部口径。
