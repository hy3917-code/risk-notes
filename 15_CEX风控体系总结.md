# CEX 风控体系设计与实践

> 基于在中心化交易所风控部门的工作经验，总结风控体系的核心模块、监控维度和处置流程。

## 一、风控架构总览

CEX 风控体系分为四大层级：

| 层级 | 职责 | 时效要求 |
|------|------|---------|
| **实时风控** | 交易链路拦截、异常订单熔断 | < 50ms |
| **准实时监控** | 行情异动、账户行为分析 | 秒级 |
| **离线分析** | 洗钱追踪、关联网络分析 | 小时/天级 |
| **事后审计** | 合规报告、监管报送 | 天/周级 |

## 二、核心风控场景

### 2.1 市场操纵检测

**检测类型：**

- **对敲交易（Wash Trading）**：同一实控账户间频繁自我成交，制造虚假交易量。通过设备指纹、IP关联、出入金路径交叉验证识别。
- **幌骗交易（Spoofing）**：大额挂单后快速撤单，诱导市场价格波动。监控撤单率 > 95% 且挂单量异常的账户。
- **分层操纵（Layering）**：多层级挂单制造虚假深度。通过挂单密度、价格分布梯度识别。

**核心指标：**
- 自成交率 > 阈值告警
- 撤单/成交比异常
- 单账户深度占比 > N%
- 关联账户群组行为一致性

### 2.2 异常交易监控

**监控维度：**

1. **频率维度**：单账户 TPS 超过基线 N 个标准差
2. **金额维度**：单笔/累计交易额触发阶梯阈值
3. **时间维度**：非活跃时段突然高频交易
4. **价格维度**：成交价偏离市场价 > 阈值
5. **对手方维度**：与标记地址/高风险地址交易

**熔断机制：**
- L1 告警：人工审核，不阻断
- L2 告警：限制提现，交易继续
- L3 告警：冻结账户，暂停所有操作

### 2.3 充提风控

| 场景 | 策略 | 参数 |
|------|------|------|
| 大额提现 | 人工审核 + 延时到账 | > 50,000 USDT |
| 新设备提现 | 24h 提现限额 | 首笔 ≤ 1,000 USDT |
| 风险地址 | 拦截 + 上报 | 链上黑名单命中 |
| 频率异常 | 滑块验证 + 冷却 | 5 次/小时 |
| 分散转出 | 关联账户聚合监控 | 多地址转同一目标 |

### 2.4 KYC/AML 合规

- **身份验证**：多因子（证件 + 人脸 + 活体检测）
- **制裁名单筛查**：OFAC / UN / 本地监管名单
- **交易监控**：大额/可疑交易自动上报（STR）
- **旅行规则（Travel Rule）**：跨 VASP 转账信息传递
- **风险评估模型**：基于行为特征的用户风险分级

## 三、技术实现要点

### 3.1 规则引擎

```
风控规则 = 触发条件 + 执行动作 + 冷却策略

条件层：
  - 原子条件：金额 > X, 频率 > Y, 时间 ∈ [A,B]
  - 组合条件：AND / OR / NOT / SEQ（时序）
  - 统计条件：滑动窗口内均值/方差/Z-score

动作层：
  - 放行 / 人工审核 / 拒绝 / 熔断 / 上报
```

### 3.2 关联网络分析

- **图数据库**（Neo4j / TigerGraph）：存储账户-设备-IP-地址关系
- **社区发现算法**（Louvain / Label Propagation）：识别团伙
- **传导风险**：标记地址 N 度邻居风险评分

### 3.3 实时数据处理

- **流处理引擎**：Flink / Kafka Streams
- **时序数据库**：InfluxDB / ClickHouse 存储指标
- **内存网格**：Redis Cluster 做实时计数器和黑名单缓存
- **回溯测试框架**：历史数据回放验证规则有效性

## 四、指标体系

### 监控大盘核心指标

| 指标 | 说明 | 告警阈值示例 |
|------|------|-------------|
| 异常交易拦截率 | 拦截量 / 异常量 | 目标 > 99.5% |
| 误拦率 | 误拦 / 总拦截 | < 0.1% |
| 规则命中率 | 命中规则的交易占比 | 监控趋势 |
| 审核时效 | 人工审核平均耗时 | < 5 min |
| 风险账户覆盖率 | 已标记/应标记 | > 95% |

## 五、经验总结

1. **规则不能太多**：初期容易堆规则，导致误拦率高、维护成本大。应优先用统计模型 + 少量核心规则。
2. **熔断要有灰度**：不要一刀切，L1/L2/L3 分级处理，既保安全又不影响用户体验。
3. **关联网络是杀手锏**：单账户行为可能正常，但放到关联网络中就能发现洗钱团伙、对敲群组。
4. **数据和日志是根基**：风控能力上限取决于数据质量。所有操作留痕，审计可追溯。
5. **与业务沟通成本**：风控不是纯技术活，需要和运营、合规、产品持续对齐策略。
