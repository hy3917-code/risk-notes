# 风控 Agent 技术 — 学习笔记

## 1. Agent vs 聊天

Agent = **想一想 → 试一下 → 看结果 → 再迭代**（ReAct）。

需要：模型、工具、状态/记忆、控制面（可观测/中断/审计）、结构化输出。

## 2. Tools

让模型能调 API/DB/文件/业务服务。  
实践：描述清晰 + input schema；工具名稳定；鉴权通过运行时注入，不暴露给模型。

## 3. 控制与安全

- 限制调用次数/时长  
- 敏感操作人工确认  
- 重试/降级  
- 全链路审计  

## 4. 记忆与上下文

对话要可恢复；太长要裁剪/摘要；状态存原始数据，用时再格式化。

## 5. 结构化输出

先定 schema；解析失败要重试/校验关键字段。

## 6. Graph（多步流程）

节点 + 显式路由；可中断恢复；错误：重试 / 等人 / 暴露异常。

## 7. 风控 PoC 架构（我画的）

- 栈：Spring Boot 3.x + Spring AI Alibaba + StateGraph  
- **SupervisorAgent**：规划步骤、调度专家、汇总  
- 专家示例：  
  - 资产分析（持仓、集中度）  
  - 行为分析（异常模式）  
  - 资金分析（充提、快进快出）  
- 每专家：ReactAgent + 专用 prompt + 工具列表 + max iterations  

## 8. 我自己的 takeaway

审核运营里重复查数、写小结最适合先 Agent 化；生产要先 mock → 只读工具 → 再写操作加审批。
