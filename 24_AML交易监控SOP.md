# 反洗钱交易监控SOP / AML Transaction Monitoring SOP

> 基于 CEX 风控实操经验，整理交易监控标准操作流程。

## 一、监控范围

| 监控对象 | 内容 | 频率 |
|---------|------|------|
| 现货交易 | 买卖方向、成交价量、手续费 | 实时 |
| 合约交易 | 开平仓、杠杆、保证金、强平 | 实时 |
| 充提币 | 地址、金额、频率、对手方 | 实时 |
| 返佣体系 | 代理关系、返佣计算、刷量检测 | 每日 |
| 新用户 | 注册后72小时行为 | 实时 |

## 二、警报分级标准

| 等级 | 定义 | 响应时间 | 处置动作 |
|------|------|---------|---------|
| L1 高风险 | 涉及大额资金、明显操纵、制裁关联 | < 5分钟 | 立即冻结 + 升级报告 |
| L2 中风险 | 异常交易模式、需人工判断 | < 30分钟 | 限制交易 + EDD审查 |
| L3 低风险 | 疑似异常、概率较低 | < 2小时 | 标记观察 + 定期复查 |

## 三、交易监控检查表

### 3.1 单笔交易检查

```
□ 交易金额是否触发大额阈值？
  → 单笔 > 50,000 USDT → L2
  → 单笔 > 200,000 USDT → L1

□ 成交价偏离市场价？
  → 偏离 > 3% → L2
  → 偏离 > 10% → L1

□ 交易对手方是否在观察名单？
  → 黑名单地址 → L1
  → 灰名单地址 → L2
```

### 3.2 账户行为检查

```
□ 单日交易频率是否异常？
  → > 100次/日 → L3
  → > 500次/日 → L2

□ 胜率/收益率是否异常？
  → 胜率 > 85% 且月收益 > 200% → L2

□ 登录设备/IP是否突变？
  → 30分钟内跨国IP切换 → L2
  → 新设备首次登录 → L3

□ 是否有历史违规记录？
  → 近30天内有违规 → 升级一级
```

### 3.3 充提币检查

```
□ 提现金额是否触发审核阈值？
  → > 10,000 USDT → L3
  → > 50,000 USDT → L2
  → > 200,000 USDT → L1

□ 提现地址是否关联风险地址？
  → 混币器地址 → L1
  → 制裁地址 → L1 立即拦截

□ 充提频率是否异常？
  → 24h内 > 10笔大额 → L2

□ 充提地址是否与其他高风险账户关联？
  → 共享充提地址 → L2
```

## 四、EDD审查流程

### 4.1 触发条件

- 交易监控命中L2及以上告警
- KYC信息存在疑点
- 用户职业/收入与交易规模不匹配
- 来自高风险地区（FATF灰/黑名单）

### 4.2 审查内容

| 审查项 | 方法 | 输出 |
|--------|------|------|
| KYC资料复核 | 证件真伪、人脸匹配度 | 通过/存疑/不通过 |
| 资金来源（SoF） | 要求提供银行流水/工资单/投资证明 | 材料齐全/不完整/拒绝提供 |
| 财富来源（SoW） | 资产证明、税务文件 | 与交易规模匹配/不匹配 |
| 交易目的 | 用户自述 vs 实际行为对比 | 一致/不一致 |
| 负面媒体 | Worldcheck / 公开信息检索 | 命中/未命中 |

### 4.3 审查结论

```
通过 → 解除限制，正常交易
有条件通过 → 设置交易限额，定期复查
升级调查 → 提交SAR，冻结账户
```

## 五、SAR撰写要点

### 5.1 报告结构

```
1. 主体信息
   - 账户ID、注册信息、KYC状态

2. 可疑活动描述
   - 时间线
   - 交易金额与频率
   - 异常行为模式

3. 调查发现
   - 账户关联关系
   - 链上资金追踪
   - KYC审查结果

4. 风险评估
   - 涉及风险类型（洗钱/欺诈/操纵）
   - 风险等级
   - 潜在影响金额

5. 处置建议
   - 建议措施（冻结/限制/监控）
   - 是否需要上报监管
```

### 5.2 写作原则

- **客观描述**：只写事实，不写猜测
- **时间线清晰**：按时间顺序排列事件
- **量化支撑**：用具体数字而非模糊描述
- **关联明确**：账户间关系需有证据支持
- **可追溯**：所有结论都有对应的调查记录

## 六、跨部门协作

| 场景 | 协作部门 | 协作内容 |
|------|---------|---------|
| 大额提现审核 | 客服部、财务部 | 确认用户身份、审核资金来源 |
| 账户冻结 | 法务部、运营部 | 法律依据确认、用户沟通话术 |
| 规则调整 | 产品部、技术部 | 风控规则上线、测试验证 |
| 监管报告 | 合规部 | SAR提交、监管沟通 |

## 七、文档与记录管理

- **案件档案**：每个L1/L2案件建立独立档案，含时间线、证据截图、调查记录、处置结果
- **日报/周报**：每日告警概况、重点案件、异常模式发现
- **特征库**：持续沉淀异常交易行为模式，支持规则迭代
- **审计轨迹**：所有操作留痕，满足内外部审计要求
