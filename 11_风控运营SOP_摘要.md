## 二、裁定标准（我的口诀）

| 术语 | 我用的判断 |
|-----|-----------|
| 秒单 | 持仓约 3 分钟内，秒单占比 >30% |
| 延时套利 | 站内 vs 外部行情延迟套利；网络延迟 >300ms 要盯 |
| 推盘 | 现货操纵带动合约；低市值、放量、合约先开仓 |
| 对敲（合约） | 多账户反向开仓平仓，刷量/体验金 |
| 对敲（现货） | 低市值转移资产；看关联、价差、是否行情推动方 |
| 伞下/同人 | 邀请下级、同设备/密码/提现地址等 |

## 三、合约 — 三类来源

1. **自动告警**：群内机器人 → 及时处理并群内反馈  
2. **客服推送**：约 **20 分钟**内排查并回复  
3. **主动监测**：按频次巡检  

### 客服推送排查（简化流程）

1. 后台 → 历史仓位 → 看币对、开平仓、盈利  
2. 是否秒单？盈利是否超阈值？→ 超阈值找**风控复核同事**  
3. 对比外部 K 线：延时套利？开仓方向是否合理？  
4. 正常则回复客服；秒单/延时套利按模板限制或升级复核  

### 限制措施（原则）

- 秒单：按持仓价值选模板；大额先复核  
- 延时套利：复核后再限制  
- 现货对敲：慎限制现货交易（客诉），重点看低市值拉盘转移  

## 四、充提 / 活动

- 充提：模式识别（快充快提、小额高频、内转）+ 地址风险  
- 活动：规则是否有漏洞；名单同人聚集  

## 五、个人工具习惯

- 后台 + 监控大盘交叉验证  
- 大额案例单独建分析文档（不对外）  
- 交班在登记表写清「未结告警」
