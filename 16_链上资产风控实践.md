# 链上资产风控实践

> 基于 STO/链上资产发行项目的风控经验，总结智能合约安全、预言机攻击防御、跨链桥监控等核心实践。

## 一、链上风控 vs CEX 风控

| 维度 | CEX 风控 | 链上风控 |
|------|---------|---------|
| 控制方式 | 中心化拦截 | 链上合约 + 链下监控 |
| 延迟要求 | 毫秒级 | 区块级（秒~分钟） |
| 可逆性 | 可回滚 | 不可逆（需治理多签） |
| 匿名性 | KYC 绑定 | 地址匿名 |
| 攻击面 | 账户/交易层 | 合约漏洞/预言机/MEV |

## 二、智能合约风控

### 2.1 部署前安全检查

```
Checklist：
□ 重入攻击防护（CEI 模式 / ReentrancyGuard）
□ 整数溢出（Solidity 0.8+ 内置 / SafeMath）
□ 访问控制（Ownable / RBAC）
□ 闪电贷攻击（价格预言机来源是否可信）
□ 前端运行（Front-running）防护
□ 时间戳依赖（block.timestamp 风险）
□ 随机数安全（不可用链上数据做随机源）
□ 治理攻击（提案-执行时间锁）
□ 升级安全（代理模式 / 多签控制）
```

### 2.2 运行时监控

**监控维度：**

1. **大额转账**：单笔 > 总供应量 X% → 告警
2. **Owner 操作**：ownership 转移 / 暂停 / 升级 → 实时通知
3. **异常调用**：未公开函数被调用 / 高频失败交易
4. **资金流向**：合约余额异动 / 异常提取
5. **Gas 异常**：某笔交易 Gas 远超正常 → 可能的 MEV 攻击

**工具链：**
- Tenderly / OpenZeppelin Defender：实时告警
- Forta Network：去中心化监控网络
- The Graph：链上数据索引查询

### 2.3 案例：闪电贷攻击防御

**攻击模式：**
1. 攻击者在单笔交易内借入大量资产（闪电贷）
2. 利用合约逻辑缺陷操纵价格
3. 套利后归还贷款，合约承受损失

**防御措施：**
- 预言机使用 TWAP（时间加权平均价格）而非瞬时价格
- 关键操作设置最小延迟（Timelock）
- 设置单笔交易滑点上限
- 接入 Chainlink / Pyth 等去中心化预言机

## 三、预言机安全

### 3.1 风险类型

| 风险 | 描述 | 案例 |
|------|------|------|
| 价格操纵 | 通过 DEX 大额交易扭曲瞬时价格 | 闪电贷 + 价格操纵 |
| 数据延迟 | 链下价格更新滞后 | 市场剧烈波动时 |
| 单点故障 | 依赖单一预言机节点 | 节点宕机 |
| 女巫攻击 | 伪造多个节点报告假数据 | 小市值代币 |

### 3.2 防御架构

```
价格获取链路：
  多数据源（CEX + DEX 聚合）
    → 中位数聚合
    → 异常值剔除（偏离中位数 > N%）
    → TWAP 平滑
    → 延迟校验（数据新鲜度）
    → 上链消费
```

## 四、跨链桥风控

### 4.1 核心风险

- **验证者攻击**：控制多签验证者 → 伪造提款证明
- **消息重放**：同一条跨链消息被多次执行
- **流动性枯竭**：单侧资产被大量提取
- **合约升级风险**：升级引入漏洞

### 4.2 监控策略

| 监控项 | 方法 | 响应 |
|--------|------|------|
| 大额跨链 | TVL 占比 > N% | 延迟处理 + 人工确认 |
| 验证者异常 | 签名集合变化 | 暂停桥 + 通知 |
| 流动性监控 | 各链资产余额 | < 阈值 → 暂停提款 |
| 消息队列堆积 | 待处理消息数 | 扩容或降级 |

## 五、链上数据分析

### 5.1 常用工具

- **Dune Analytics**：SQL 查询链上数据，制作 Dashboard
- **Nansen**：标记地址、Smart Money 追踪
- **Chainalysis / TRM Labs**：合规级链上分析
- **Etherscan API**：基础交易查询和合约验证

### 5.2 关键分析场景

1. **代币持有集中度**：前 10 地址持币占比
2. **关联地址发现**：资金流图分析
3. **异常价格发现**：DEX 价格 vs CEX 价格价差
4. **合约交互热力图**：识别高频/异常交互模式
5. **MEV 提取分析**：Sandwich / Arbitrage 行为识别

## 六、实践经验

1. **链上风控的核心是"监控 + 响应"**：无法像 CEX 一样实时拦截，所以早期发现 + 快速多签响应至关重要。
2. **预言机是阿喀琉斯之踵**：90% 的 DeFi 攻击涉及价格操纵，TWAP + 多源聚合是最低配置。
3. **工具链比代码更重要**：Forta 告警 + Tenderly 模拟 + Dune 分析构成完整链路。
4. **治理安全不容忽视**：多签私钥管理、时间锁参数、提案门槛都直接决定协议安全性。
5. **跨链安全是未来重点**：2022-2024 跨链桥被盗金额超 20 亿美元，桥的风控不应低于链本身。
