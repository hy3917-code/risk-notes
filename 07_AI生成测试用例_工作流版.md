# 用 Coze 工作流自动生成测试用例（我的操作笔记）

## 目标

上传需求文档 → AI 生成结构化用例 → 可选转思维导图，提高用例设计效率。

## 环境

- Coze 平台账号  
- 支持：截图、Word、PDF、TXT  
- 需求最好用用户故事 / 功能说明 / 流程图+文字  

## 节点搭建（我的流程）

1. **开始节点**：变量名好辨认；类型选 Doc 上传；多格式就多变量，**尽量不要全必填**。  
2. **读取文档插件**：把开始节点的文件传给插件。  
3. **大模型节点**：输入取插件输出；模型选能力强的；输出类型选字符串（MD）。  
4. **MD 转思维导图插件**：搜索「生成思维导图」类插件；`markdown` 引用上一步输出。  
5. **结束节点**：输出思维导图链接。

## 限制（我自己会注意的）

- 模糊需求生成质量差，**必须人工评审**  
- 一般覆盖 70–80% 明显场景，边界/集成要手补  
- 不能替代探索性测试  

## FAQ 自记

- 性能用例：能出场景和验证点，压测脚本另写  
- 提高质量：需求写清范围、约束、参考用例；分模块生成
