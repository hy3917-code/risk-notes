# Playwright CLI + Claude：低 Token 自动化测试（个人总结）

## 想法

浏览器自动化（Playwright）负责「手脚」，AI 只在需要时做「脑子」— 理解任务、拆步骤、编排 Skill，**Token 尽量为 0**。

## 四层架构

| 层 | 内容 |
|----|------|
| 准备 | Node.js、Playwright、Chrome |
| 能力 Skill | 打开需求、学业务、生成用例、按用例点验、出报告 |
| 任务 | 多个 Skill 串成完整流程 |
| AI（可选） | 任务理解、拆解、生成调用链 |

```
AI 理解 → 执行流程 → 调 Skill → Playwright → 结果
```

## 三个原则

1. **能不用 AI 就不用** — 稳定、便宜  
2. **Skill 化** — `login()` / `search()` 可复用可组合  
3. **CLI 优先** — 可控、可复现、好调试  

## 我做过的一例

- 需求：审核中心「提现审核」新增导出  
- 步骤：读需求 → 生成用例 → 在**测试环境风控后台**点验 → 汇总结果  
- 待解决：验证码自动输入  

## 定位

不是「纯 AI 自动化」，而是 **自动化 + AI 增强**。
