# CEX异常交易案例手册 / CEX Abnormal Trading Casebook

> 基于 Jucom CEX 风控实操经验，整理常见异常交易类型、识别特征与处置方法。

## 一、对敲交易（Wash Trading）

**行为特征：**
- 相同或关联账户之间频繁相互成交
- 成交价格偏离市场价极小（< 0.1%）
- 交易量巨大但持仓不变
- 常在低流动性交易对出现

**识别方法：**
| 维度 | 信号 |
|------|------|
| 账户关联 | 同设备指纹、同IP段、同出入金地址 |
| 交易模式 | 买卖方向相反、成交时间接近（< 5s） |
| 量价关系 | 成交量放大但K线波动极小 |
| 手续费消耗 | 频繁交易产生高额手续费，不符合经济理性 |

**处置建议：**
1. 确认关联关系后冻结关联账户群
2. 回溯清洗该交易对虚假成交量
3. 将设备指纹和IP加入黑名单库
4. 调整该交易对手续费率（提高对敲成本）

## 二、分层套利（Layering Arbitrage）

**行为特征：**
- 单用户或关联群组在多个价格层级挂单
- 利用价差和流动性深度进行无风险套利
- 常见于新币上线初期或低流动性时段

**识别方法：**
| 维度 | 信号 |
|------|------|
| 挂单行为 | 多层级限价单，覆盖±N%价格区间 |
| 撤单率 | 撤单率 > 90%，仅成交有利方向 |
| 多账户协同 | 不同账户在不同层级互补挂单 |
| 时间窗口 | 集中在低流动性时段（如亚洲凌晨） |

**处置建议：**
1. 限制单用户挂单层级数量
2. 对高频挂撤单账户提高手续费
3. 识别关联账户群后统一管控

## 三、多账户协同操纵

**行为特征：**
- 3个以上账户协同操作同一交易对
- 有人负责拉升/打压，有人负责出货/吸筹
- 常伴随社交媒体配合（喊单群）

**识别方法——多维关联图谱：**

| 维度 | 分析方法 |
|------|---------|
| 设备指纹 | 同设备登录多个账户 |
| IP聚集 | 同IP/C段多个账户集中交易 |
| 出入金关联 | 资金归集到相同提现地址 |
| 交易对手关联 | 频繁互为交易对手 |
| 行为时序 | 操作时间高度同步（秒级） |
| 链上地址图谱 | 资金链路呈现星形/环形结构 |

**案例：异常攻击团伙伴案**

在一次日常监控中发现：5个账户在30分钟内集中交易某小市值币种，3个账户持续买入推高价格，2个账户在高位分批卖出。深入调查发现：
- 所有账户使用同一设备型号和浏览器指纹
- IP地址集中在同一城市同一运营商段
- 链上充币地址均来自同一交易所同一批次
- 最终盈利资金归集到同一OTC地址

处置：冻结5个关联账户，封禁设备指纹，将该操纵模式加入规则引擎。

## 四、手续费套利（Fee Arbitrage）

**行为特征：**
- 利用做市商/VIP费率优势进行高频低利润套利
- 通过返佣机制刷取平台激励
- 虚假交易刷交易量获取手续费返还

**识别方法：**
| 维度 | 信号 |
|------|------|
| 盈亏特征 | 扣除手续费前微利，扣除后亏损 |
| 交易频率 | 远超正常交易频率（>100次/小时） |
| 对手方 | 自成交或固定对手方循环交易 |
| 返佣关联 | 代理与被代理账户间存在循环交易 |

**返佣体系风控要点：**
1. 检测代理层级异常（自返佣、多层嵌套）
2. 识别虚假交易量（对敲刷量）
3. 监控代理与被代理方的IP/设备关联
4. 设置返佣上限和冷静期

## 五、高盈利账户专项筛查

**筛查维度模型：**

| 维度 | 指标 | 阈值参考 |
|------|------|---------|
| 胜率异常 | 盈利交易占比 | > 85% |
| 收益率异常 | 累计收益率 | > 500% / 月 |
| 交易频率 | 日均开平仓次数 | > 50次 |
| 品种集中度 | 单一币种交易占比 | > 80% |
| 持仓时间 | 平均持仓时长 | < 5分钟 |
| 流动性偏好 | 低流动性币种交易量占比 | > 60% |

**处置流程：**
1. 命中3项以上 → 列入重点关注名单
2. 人工审核交易记录和K线对应关系
3. 确认异常 → 限制交易权限 + 提现审核
4. 输出案例报告，沉淀到特征库
