# 风控侧「资金对账」需求调研（个人起草）

> 已去掉内部对接人姓名、具体系统代号。

## 背景

各业务系统目标与风险容忍度不一致，缺少**统一的资金安全监控视角**。计划建设**独立于业务记账**的对账系统，做持续校验与风险分级。

## 目的

1. 监控用户资金异常  
2. 监控账实一致性  
3. 为风控决策提供依据  

## 三个核心问题

1. **钱对不对？** — 各系统账 vs 实际资金  
2. **不一致是否正常？** — 业务/会计能否解释  
3. **危不危险？** — 是否升级、是否介入  

资金链路（我画的）：

```
链上 → 钱包资产账 → 交易所用户/平台账 → 财务账
```

## 调研对象与要点

### 钱包团队

- 提币放行如何判断「能提」？大额高频阈值？  
- 钱包账 vs 链上不一致：已知可接受多久？  
- 盲区：归集延迟、失败重试、矿工费、地址调拨  

### 交易所团队

- 余额变动场景是否都有明细（返佣、充提、买卖、活动、调账等）？  
- 多业务线共用资金、撮合清算延迟的风险点？  
- 「账平但钱不安全」有没有历史 near-miss？  

### 财务团队

- 会计上正常、实时风控上异常的信号？  
- 与钱包/交易所对接方式？对外提币是否经审核？  

## 系统定位（我主张的）

- **独立**于钱包 / 交易所 / 财务  
- 只做：校验、监控、判断 — **不参与业务记账**

```
各系统数据 → 对账模型 → 差异识别 → 风险分级 → 规则/策略
```

## 规则思路（初稿）

- 差异金额、持续时间  
- 大额/高频异动  
- 跨系统不一致  

输出：告警、限制建议、事件升级。

## 会议期望产出

1. 数据清单与来源  
2. 差异类型统一认知  
3. 一期范围  
4. 产品/研发路线图
